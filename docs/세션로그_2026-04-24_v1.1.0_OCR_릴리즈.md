## 이번 세션에서 한 일

### 1. PrintScreen 독점 사용 옵트인 토글
기존 코드가 사용자 동의 없이 `HKCU\Control Panel\Keyboard\PrintScreenKeyForSnippingEnabled`를 조용히 0으로 덮어쓰고 있었음 (MainWindow에 하드코딩).

- 신규 `Services/PrintScreenOverrideService.cs` — 레지스트리 쓰기/복원 격리
- `HotkeySettings.OverridePrintScreen` 추가 (기본 true)
- 설정 UI 체크박스 + 툴팁 (단축키 탭)
- 토글 OFF 시 레지스트리 `1`로 복원 → Windows 기본 Snipping Tool 동작

### 2. OCR 기능 (Windows.Media.Ocr 기반)
**설계 결정**:
- 엔진 후보 A/B/C/D 중 **A (Windows 내장)** 선택
  - 오프라인, 설치 용량 증가 0MB, SignPath 영향 0
  - 트레이드오프: 손글씨/왜곡 텍스트는 AI OCR(PaddleOCR 등)보다 약함
- TargetFramework `net8.0-windows` → `net8.0-windows10.0.19041.0` 필수

**구현 파일**:
- `Services/OcrService.cs` (신규) — 정적 클래스, WinRT 완전 격리
  - `ExtractTextAsync(Bitmap, lang, preprocess)` → string
  - `GetAvailableLanguages()` / `IsLanguageAvailable(tag)`
  - `ResolveLanguageOrFallback(preferred)` — 공통 폴백 체인 (설정→ko→en-US→첫 설치)
- `Models/CaptureMode.cs` — enum에 `Text` 추가
- `Models/AppSettings.cs` — `HotkeySettings.TextCapture` + 신규 `OcrSettings` 클래스
- `Views/MainWindow.xaml.cs` — TextCapture 핫키 + `RunOcrAndNotify` (토스트)
- `Views/EditorWindow.xaml(.cs)` — 툴바 OCR 버튼 + 하단 OCR 패널

**2가지 진입점**:
1. **빠른 플로우**: `Ctrl+Shift+T` → 영역 드래그 → OCR → 클립보드 + NotifyIcon 토스트 (40자 미리보기)
2. **정밀 플로우**: 편집기 툴바 🔤 버튼 → 하단 패널 펼침 → TextBox 편집 → 복사

**테스트** (tests/ShinCapture.Tests/Services/OcrServiceTests.cs):
- 처음엔 en-US 하드코딩이었으나 한국어 팩만 있는 로컬 환경에서 실패 발견 → **환경 비의존적 테스트**로 리팩터 (`GetAvailableLanguages().First()` 사용)
- 8개 테스트 추가, 최종 29/29 통과

### 3. OCR 인식률 향상 전처리
기본 엔진 그대로 두고 입력 이미지 전처리만 개선 (옵션 A 접근):

- **공격적 업스케일**: 기존 40px 기준 2x만 → 새로 <50px→4x, <100px→3x, <200px→2x
- **어두운 배경 자동 반전**: 격자 샘플링으로 평균 명도 <128 판단 → ColorMatrix 반전 후 OCR
- 원본 비트맵은 건드리지 않음 (임시 사본에만 전처리, 사용 후 Dispose)
- `OcrSettings.UpscaleSmallImages`로 토글 가능 (기본 ON)

### 4. UI 개선
- **OCR 버튼 강조**: 기존에는 드로잉 도구 배열에 끼어있고 `if (name == "OCR")` branch hack (리뷰어 지적). 개선: 도구 배열에서 분리 → 별도 `CreateOcrActionButton()` → accent 파란색 배경 + "텍스트 추출" 라벨 + 구분선으로 분리
- **새 캡쳐 시 이전 OCR 결과 자동 숨김**: `LoadNewCapture()`에 `OcrPanel.Visibility = Collapsed` + 텍스트박스 초기화 추가
