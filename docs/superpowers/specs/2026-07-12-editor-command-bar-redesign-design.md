# 신캡쳐 편집창 명령바 재설계

## 목표

기존 편집 기능과 단축키를 유지하면서 메인 편집창의 명령 구조, 시각 언어, 적응형 배치, 접근성을 정돈한다. 사용자는 캡처 직후 편집 도구와 저장 동작을 빠르게 찾을 수 있어야 하며, 창 크기를 바꿔도 툴바가 불규칙하게 여러 줄로 접히지 않아야 한다.

## 디자인 방향

- Windows 데스크톱 생산성 도구에 맞는 조용하고 선명한 Soft UI를 사용한다.
- `Segoe UI Variable`과 기존 인디고 포인트 색을 유지해 제품 정체성을 보존한다.
- 트레이 플라이아웃의 1.8px 벡터 아이콘, 포커스 링, 상태 표현을 편집창에도 적용한다.
- 장식은 줄이고 캡처 이미지가 항상 화면의 주인공이 되도록 한다.

## 명령 구조

상단은 한 줄 명령바로 고정한다.

1. 편집 도구: 선택, 펜, 형광펜, 도형, 화살표, 텍스트, 말풍선, 모자이크, 블러, 번호, 이미지, 색상 추출, 크롭, 지우개
2. AI 도구: 텍스트 추출, 번역 캡처, 스마트 컷을 하나의 `AI 도구` 메뉴로 묶는다.
3. 히스토리 명령: 실행취소와 다시실행을 편집 도구 뒤에 둔다.
4. 파일 명령: 복사, 다른 이름으로 저장, 저장을 오른쪽에 둔다.
5. 보조 명령: 설정과 API 키 안내는 더보기 메뉴로 이동한다.

광고는 편집 명령바, 캡처 기록, 상태바에서 제거한다. 홍보 링크는 도움말 또는 정보 화면에서만 제공한다.

## 적응형 배치

- `Comfortable`(1500 DIP 이상): 도구 아이콘과 라벨, 캡처 기록 패널을 모두 표시한다.
- `Compact`(850~1499 DIP): 편집 도구 라벨을 숨기고 아이콘과 도구 설명을 유지한다.
- `Narrow`(850 DIP 미만): 아이콘 모드를 유지하고 캡처 기록 패널을 접을 수 있게 한다.
- 어떤 모드에서도 상단 명령바는 `WrapPanel`로 여러 줄이 되지 않는다. 넘치는 도구는 가로 스크롤 또는 더보기 메뉴에서 접근할 수 있어야 한다.

## 속성 패널

- 현재 선택 도구에 필요한 속성만 표시하는 기존 모델을 유지한다.
- 색상 견본의 시각 크기를 18px, 클릭 영역을 최소 28px로 키운다.
- 마우스 이벤트 전용 `Border` 대신 키보드 포커스가 가능한 버튼을 사용한다.
- 고급 색상은 별도 색상 선택 버튼으로 유지한다.
- 속성 영역은 일정한 최소 높이를 유지해 도구를 바꿀 때 캔버스가 크게 흔들리지 않게 한다.

## 캡처 기록

- 패널 폭을 180px로 늘린다.
- 헤더 문구는 `캡처 기록`, `전체 저장`, `기록 비우기`로 정리한다.
- 선택된 썸네일에는 인디고 테두리와 번호·해상도 정보를 함께 표시한다.
- 우클릭 메뉴는 편집창 테마의 모서리, 간격, 포커스 상태를 사용한다.
- 기존 위/아래 방향키 이동과 자동화 이름을 유지한다.

## OCR 패널과 상태바

- OCR 원문과 번역문은 `Consolas` 대신 앱 기본 글꼴을 사용한다.
- 이모지를 구조 아이콘으로 사용하지 않는다.
- 닫기·복사·번역 버튼은 명확한 포커스와 비활성 상태를 제공한다.
- 상태바에는 이미지 크기, 포맷, 줌, 작업 상태만 둔다.
- 줌에는 축소, 100%, 확대, 화면 맞춤 동작을 제공한다.

## 접근성

- 모든 도구와 아이콘 버튼에 `AutomationProperties.Name`을 제공한다.
- 키보드 포커스가 시각적으로 보이고 Tab 순서가 화면 순서와 일치해야 한다.
- 선택 상태는 색뿐 아니라 테두리 또는 표시선으로도 구분한다.
- 기존 도구 단축키와 저장 단축키를 유지한다.

## 검증

- 적응형 레이아웃 정책과 아이콘 키 목록은 단위 테스트로 검증한다.
- 전체 테스트와 Release 빌드를 실행한다.
- 760×520, 1100×750, 최대화 상태에서 메뉴 줄바꿈, 캔버스 크기, 기록 패널을 시각적으로 확인한다.

## 제외 범위

- 편집 도구의 그리기 알고리즘 변경
- 캡처 방식 또는 저장 포맷 변경
- 다크 모드 신규 구현
- OCR 및 번역 서비스 로직 변경
