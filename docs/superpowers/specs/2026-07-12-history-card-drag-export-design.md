# 캡처 기록 카드 드래그 내보내기 설계

## 목표

편집창 우측 `캡처 기록`의 각 카드를 좌클릭으로 끌어 터미널에는 PNG 파일 경로를, 카카오톡·디스코드·메일 등에는 이미지 파일을 전달한다. 별도 상단 버튼이나 새 창은 추가하지 않는다.

## 상호작용

- 카드에서 좌클릭 후 시스템 최소 드래그 거리 미만으로 놓으면 기존처럼 해당 캡처를 연다.
- 최소 드래그 거리를 넘으면 클릭 동작을 취소하고 파일 복사 드래그를 시작한다.
- 우클릭 메뉴, 키보드 위·아래 이동, 포커스 이동은 기존 동작을 유지한다.
- 카드 도구 설명과 접근성 도움말에 `클릭: 열기 · 드래그: 다른 앱으로 보내기`를 표시한다.
- 드래그 중 카드는 살짝 흐려지고, 완료·취소·실패 결과는 상태 표시줄에 알린다.

## 편집 결과 합성

드래그 시작 직전에 현재 캡처의 편집 개체를 저장한다. 드래그한 카드가 현재 이미지가 아니어도 `_captureObjects`에 보존된 모자이크·도형·텍스트 등의 편집 상태를 원본과 합성한다. 기존 `RenderFinalImage`는 임의의 `BitmapSource`와 편집 개체 목록을 받을 수 있는 공통 렌더러로 분리한다.

## 임시 파일과 드래그 데이터

- 저장 위치: `%LOCALAPPDATA%\ShinCapture\Temp\DragDrop`
- 파일명: `ShinCapture_yyyyMMdd_HHmmss_fff_<suffix>.png`
- `.tmp`로 먼저 기록한 뒤 최종 PNG로 이동해 부분 파일 노출을 막는다.
- WPF `DataObject`에 `FileDrop`과 `Bitmap`을 함께 넣고 `DragDropEffects.Copy`로 전달한다.
- 드롭 직후 삭제하지 않는다. 대상 앱이 비동기로 파일을 읽을 수 있기 때문이다.

## 정리 정책

드래그 내보내기 서비스 생성 시와 새 파일 생성 전에 정리한다. 24시간이 지난 파일을 삭제하고, 남은 파일이 100개 또는 250MB를 넘으면 오래된 파일부터 제거한다. 잠긴 파일이나 삭제 실패는 드래그 기능을 중단시키지 않는다.

## 검증

- xUnit으로 고유 PNG 생성, 원자적 저장, 24시간 만료, 개수·용량 제한을 검증한다.
- 합성 렌더러가 현재 카드와 비현재 카드의 편집 개체를 사용하는지 코드 수준에서 검증한다.
- 전체 Release 테스트와 빌드를 실행한다.
- Windows Terminal, PowerShell, CMD에는 경로가 입력되고 카카오톡에는 이미지가 첨부되는지 수동 검수한다.
